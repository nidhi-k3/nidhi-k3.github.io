Matchamatic IoT Drink Maker

Automated matcha preparation system reducing drink-making time from 15 minutes to 7 minutes

Role
Hardware & Software Engineer
Course
ECE 655: Full-Stack IoT Systems
Team
2 people

The Problem

Making a perfect matcha latte is time-consuming: 15-20 minutes of soaking the whisk, sifting powder, whisking, frothing milk, and assembling ingredients. We wanted to automate the tedious parts while keeping the ritual enjoyable.

The Solution

Matchamatic is an ESP32-based automated drink dispensing system that controls peristaltic pumps for precise liquid dispensing and DC motors for whisking and frothing. Users select their preferences on a web dashboard, and the system handles the rest.

Hardware Design

  • Microcontroller: ESP32 with Wi-Fi for web server hosting
  • Motor Control: 4-channel relay module controlling peristaltic pumps; MOSFET voltage step-up circuit to drive dual DC motors
  • Mechanical Design: 3D-printed motor mounts and whisk modules for precise positioning
  • Power System: External power supplies for pumps and motors; regulated 5V for ESP32
  • Enclosure: Custom wood housing with organized bottle storage

Software Architecture

Backend: MicroPython-based web server running on ESP32, managing GPIO control logic for precise liquid dispensing and motor actuation

Frontend: Flask + HTML/CSS web dashboard where users select sweetness level and cold foam preferences

Control Logic: Timed pump activation for measured dispensing; motor speed control for whisking matcha and frothing cream

Circuit Design & Power Management

One of the key challenges was power management. The peristaltic pumps and DC motors required different voltage levels and higher current than the ESP32 could provide directly. We designed a MOSFET-based voltage step-up circuit to safely drive the motors while protecting the microcontroller.

The 4-channel relay module acted as an isolation barrier, allowing low-power GPIO signals from the ESP32 to control high-current pump circuits.

Key Features

  • Automated dispensing of water, milk, cold foam, and syrup with timed precision
  • Dual motor system for whisking matcha and frothing cream
  • Web-based interface accessible via WiFi—no app installation needed
  • Modular design allowing easy maintenance and ingredient refills

Development Process

We started with breadboard prototyping to test pump flow rates and motor speeds. Once we validated the concept, we moved to a more permanent assembly with soldered connections and organized wiring.

Testing revealed that cold foam pump performance was inconsistent: we ended up making it a semi-automated step where the user manually pours cream, and the system froths it. This taught me the importance of managing scope and focusing on what adds the most value.

Results

The final system successfully reduced matcha preparation time by 50%, from 15 minutes to 7 minutes. The web interface made customization easy, and the automated dispensing was consistent and accurate.

This project gave me hands-on experience with full-stack IoT development, from circuit design and motor control to web server programming and user interface design.

Future Improvements

  • Add refrigeration to keep milk and creamer cold
  • Implement weight sensors for matcha powder to dynamically adjust water/milk ratios
  • Design a more stable whisking mechanism to reduce vibration
