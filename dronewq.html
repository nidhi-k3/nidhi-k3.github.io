Project Overview

DroneWQ is a desktop application built with Python and Electron that processes multispectral drone images to analyze water quality metrics. The tool reduces manual data processing time from 8 hours to 15 minutes—a 97% reduction—by automating image analysis, environmental metrics calculation, and report generation.

My Contributions

  • Package Distribution: Deployed PyPI package to make core processing modules reusable across research projects
  • Accessibility Features: Integrated Google Gemini API to generate AI-powered plot descriptions for screen reader accessibility
  • Keyboard Navigation: Implemented comprehensive keyboard shortcuts and focus management for improved accessibility compliance

Accessibility Implementation

One of my favorite features to implement was the AI-generated plot descriptions. Many researchers rely on screen readers, and traditional data visualizations are inaccessible to them. I integrated Google's Gemini API to automatically generate descriptive text for each plot, explaining trends, outliers, and key findings.

I also implemented comprehensive keyboard navigation, ensuring every feature in the app is accessible without a mouse. This included proper focus management, ARIA labels, and semantic HTML structure.

Data Processing Pipeline

The core challenge was efficiently processing large batches of multispectral imagery from MicaSense RedEdge-MX sensors. Each drone flight produces hundreds of images across 5 spectral bands (blue, green, red, red edge, NIR). We refactored the codebase into an object-oriented architecture that:

  • Processes raw imagery to remote sensing reflectance (Rrs) using Hedley glint removal and DLS downwelling irradiance
  • Applies pixel masking with NIR thresholding to filter glint and invalid pixels
  • Calculates chlorophyll-a concentrations using NASA's blended Hu OCX algorithm
  • Computes total suspended matter (TSM) using the Nechad algorithm
  • Georeferences images based on GPS, altitude, and sensor orientation (yaw, pitch, roll)
  • Mosaics georeferenced images into cohesive maps of water quality parameters

Impact

DroneWQ has been used by environmental researchers to analyze water quality in lakes and reservoirs. The 97% reduction in processing time means researchers can analyze more datasets, run more experiments, and spend their time on insights rather than manual data wrangling.

The PyPI package deployment makes the core algorithms reusable for other research groups, extending the project's impact beyond our immediate team.

Technical Challenges

  • Performance Optimization: Processing 1000+ images efficiently required careful memory management and parallel processing
  • API Design: Creating a clean, intuitive Flask API that the frontend could easily consume
  • Error Handling: Robust error handling for corrupted images, missing metadata, and network failures

Lessons Learned

  • The importance of clean API design. Well-documented endpoints make frontend integration smooth
  • Accessibility should be built in from the start, not added as an afterthought
  • Package distribution (PyPI) makes code reusable and increases project impact
  • User feedback is critical... researchers had specific workflow needs we hadn't anticipated
